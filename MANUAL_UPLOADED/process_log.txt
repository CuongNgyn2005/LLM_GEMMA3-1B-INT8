Address decisions
- VPU/IP physical base: 0xA0000000.
- Vivado address editor range: 0xA0000000 - 0xAFFFFFFF.
- Project reserved DDR range: 0x70000000 - 0x7FFFFFFF.
- Current DATN_RTL RTL is an AXI4-Full slave with internal BRAM windows, so the host writes data directly to IP windows under 0xA0000000 instead of passing DDR buffer addresses to the IP.

Implemented host flow
1. Open /dev/mem with O_RDWR | O_SYNC.
2. mmap VPU base 0xA0000000.
3. Read VPU limits at local offset 0x70 when available.
4. For each accepted llama.cpp Q8_0 x F32 matmul:
   - quantize activation F32 to q8_0 blocks of 32 values;
   - for each output-row tile of up to 128 rows;
   - for each K block of 32 values;
   - write activation to local offset 0x00010000;
   - write weights to local offset 0x00100000 using row * MAX_COL_BEATS + beat;
   - write ROWS/COLS/COL_BEATS/SCALE/MODE registers;
   - start at CTRL offset 0x00;
   - poll STATUS offset 0x10 for done/error;
   - read INT32 partials from local offset 0x00200000;
   - multiply partials by activation and weight q8_0 scales in software and accumulate F32 output.

Known fallback behavior
- Attention/FLASH_ATTN_EXT is not sent to FPGA, because bitstream_matrix.bit exposes GEMV only and has no KV-cache/attention register path.
- Invalid shapes or unsupported tensor types return to the normal CPU path.
- If a hardware timeout/error happens after a shape was accepted, thread 0 computes a local q8_0 software fallback so worker threads that already returned do not leave dst uncomputed.
